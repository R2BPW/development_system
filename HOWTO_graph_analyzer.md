# Как запустить граф-аналитик

## TL;DR

```
/запустить граф-аналитик /путь/к/граф.md /путь/к/модели.py
```

или прислать `.md`-файл в Telegram и нажать «▶️ Анализировать».

---

## Что нужно подготовить

### 1. Граф процесса — `граф.md`

Mermaid-файл с `flowchart TD`. Требования:
- Узлы с метками в кавычках: `A0["E1 Текст действия"]`
- Ветки решений: `A3{"Условие?"}`
- Error-узлы с `ERR` в имени: `ERR1["Компенсация"]`
- Рёбра с условиями: `A3 -->|"Да"| A4`

Для `add_tour` граф уже лежит здесь:
```
/root/.openclaw/media/inbound/add_tour_process_mermaid---.md
```

### 2. Django-модели — `модели.py`

Можно собрать вручную или скриптом (см. ниже).
Достаточно только тех классов, которые участвуют в процессе.
Для `add_tour`:

```bash
python3 /root/.openclaw/workspace/development_system/tools/extract_models.py \
  WasteSiteVisitedContainers AddTourTaskExecutionLog RoutePlanningCalculation \
  > /tmp/models_add_tour.py
```

---

## Запуск через мастер-бот

```
/запустить граф-аналитик <граф.md> <модели.py>
```

Повторить последний запуск (если граф/модели не изменились):
```
/запустить граф-аналитик повторить
```

Последние пути сохраняются в `/tmp/граф-аналитик-последний.lisp`.

**Требуется:** `OPENROUTER_API_KEY` в окружении мастера.
**Модель:** `ANALYZER_MODEL` (по умолчанию `anthropic/claude-opus-4-5`).

---

## Запуск напрямую через SBCL (без мастера)

```bash
OPENROUTER_API_KEY="sk-or-v1-..." \
sbcl --noinform \
  --eval '(load "/root/quicklisp/setup.lisp")' \
  --eval '(ql:quickload (quote ("cl-ppcre" "cl-json" "dexador")) :silent t)' \
  --load /root/.openclaw/workspace/development_system/мастер/потоки/граф-аналитик.lisp \
  --eval '
(in-package :поток-граф-аналитик)
(setf *макс-символов* 99999)
(format t "~A~%" (выполнить-с-файлом "/tmp/граф.md" "/tmp/модели.py"))
' \
  --eval '(sb-ext:exit)'
```

## Что спросить у ассистента

Когда обновил код и хочешь перезапустить:

> «Перезапусти граф-аналитик для add_tour.
> Граф: [файл или "тот же"].
> Модели обновились — перечитай из apro-new классы X, Y, Z.
> Сделай PDF-отчёт.»

Ассистент сам: вытащит модели из кода, запустит анализ, сгенерирует PDF.

---

## Как обновить модели при изменении кода

```bash
# Извлечь конкретные классы из репозитория
python3 /root/.openclaw/workspace/development_system/tools/extract_models.py \
  WasteSiteVisitedContainers AddTourTaskExecutionLog RoutePlanningCalculation \
  > /tmp/models_add_tour.py

# Затем запустить анализ
/запустить граф-аналитик /tmp/add_tour.md /tmp/models_add_tour.py
```

---

## Как поменять промпты или добавить шаг анализа

Редактировать напрямую:
```
development_system/мастер/потоки/граф-аналитик.lisp
```

Параметр `*промпты*` — список `(:ключ "шаблон")`.
Плейсхолдеры: `{ГРАФ}`, `{МОДЕЛИ}`, `{ОШИБКИ}`, `{НЕДОСТИЖИМЫЕ}`.

После правки поток перезагружается автоматически при следующем вызове.

## Формат отчёта

Всегда: только находки, без введений. Три раздела:
1. Критичные проблемы (🔴)
2. Предупреждения (⚠️)
3. Замечания (ℹ️)

Запросить PDF: добавить в конце «сделай PDF».
