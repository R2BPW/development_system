# development_system

Двухуровневая система Telegram-ботов для порождения и исполнения вычислительных потоков.

## Участники

| Бот | Язык | Роль |
|-----|------|------|
| **Мастер** | Racket | Обсуждает задачи, порождает потоки через Claude Code |
| **Подмастерье** | Racket + SBCL | Принимает задачу, прогоняет через активный поток, возвращает ответ |

Потоки — самодостаточные файлы на Common Lisp с точкой входа `(выполнить задача)`.

## Запуск

```bash
export MASTER_BOT_TOKEN=<токен Мастера>
export APPRENTICE_BOT_TOKEN=<токен Подмастерья>
export OPENROUTER_API_KEY=<ключ OpenRouter>

nohup racket мастер/main.rkt >> /tmp/master.log 2>&1 &
nohup racket подмастерье/main.rkt >> /tmp/apprentice.log 2>&1 &
```

## Команды Мастера

```
/породить <описание>          — создать новый поток через Claude Code
/потоки                       — список потоков
/запустить <имя>              — активировать поток для Подмастерья
/остановить                   — остановить активный поток
/переключить <имя>            — сменить поток
/цепочка <a> <b>              — последовательное комбинирование
/параллель <a> <b>            — параллельное выполнение
/ветвление <условие> <a> <b>  — условный выбор потока
/комбинировать <описание>     — новый способ через Claude Code
/душа                         — конфигурация
/память                       — долговременная память
/состояние                    — активный поток и время работы
/время                        — время сервера
```

Свободный текст — диалог с моделью для обсуждения задачи.

## Структура

```
мастер/
  main.rkt            — бот-оркестратор
  dusha.rkt           — чтение/запись конфигурации и истории
  potoki.rkt          — порождение, проверка, запуск потоков
  kombinirovanie.rkt  — комбинирование потоков
  llm.rkt             — обращение к OpenRouter
  душа.scm            — конфигурация (модель, девиз, память)
  потоки/             — порождённые .lisp и .мета файлы
  история/            — журнал диалогов по дням

подмастерье/
  main.rkt            — бот-исполнитель
  исполнитель.lisp    — загрузка потока, вызов SBCL, condition/restart
```
