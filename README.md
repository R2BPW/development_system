# Development System

Система для работы с вычислительными потоками через Telegram-бот.
Потоки — это Common Lisp модули с одной точкой входа. Мастер загружает их в живой образ и вызывает по команде.

## Как это работает

```
Пользователь → Telegram → Мастер (SBCL)
                               │
                    load / funcall выполнить
                               │
                          Поток (*.lisp)
                               │
                       LLM / Python / etc.
```

**Мастер** — Telegram-бот. Принимает команды, загружает потоки в образ, вызывает `выполнить`, хранит историю диалога.

**Поток** — файл `*.lisp` с одним пакетом `:поток-ИМЯ` и одной функцией `(выполнить задача) → строка`.

**Порождатель** — метапоток: принимает описание на русском, просит LLM написать код нового потока, компилирует и загружает его в тот же образ без перезапуска.

## Команды

| Команда | Описание |
|---|---|
| `/старт` | Показать меню |
| `/породить <описание>` | Сгенерировать новый поток через LLM |
| `/запустить <имя> <задача>` | Выполнить задачу через поток |
| `/потоки` | Список загруженных потоков |
| `/диалог <текст>` | Диалог с LLM с историей |
| `/сбросить` | Сбросить историю диалога |
| Свободный текст | Перенаправляется в `/диалог` |

## Потоки

| Поток | Что делает |
|---|---|
| `эхо` | Возвращает задачу как есть |
| `уточнение` | Уточняет задачу через LLM |
| `исполнитель` | Выполняет задачу через LLM |
| `питон` | LLM генерирует Python-код, запускает, возвращает вывод |
| `выполни-код` | То же, с явным извлечением ```python блока |
| `реакт` | ReAct-агент: рассуждение + арифметика |
| `порождатель` | Порождает новые потоки через LLM |
| `агент-аналитик` | Агентный анализ кода: LLM через tool calls исследует граф процесса + репозиторий |

## Запуск

```bash
export MASTER_BOT_TOKEN="..."
export ADMIN_CHAT_ID="250203350"
export OPENROUTER_API_KEY="sk-or-v1-..."

sbcl --load ~/quicklisp/setup.lisp \
     --eval '(push #p"мастер/cl/" asdf:*central-registry*)' \
     --eval '(asdf:load-system :мастер :verbose nil)' \
     --eval '(мастер:start)'
```

Текущий offset сохраняется в `/tmp/cl-master-offset.txt` и восстанавливается при перезапуске.


## Ключевые свойства

- **Без перезапуска** — новый поток компилируется и загружается в живой образ через `compile-file` + `load`
- **Обнаружение через рефлексию** — `find-package` + `find-symbol` + `fboundp`, никаких реестров
- **Однопоточность** — нет race conditions, нет мьютексов; зависший поток блокирует только обработку текущего сообщения
- **Метациркулярность** — `порождатель` — сам поток, генерирует потоки; слой генерации = слой исполнения
