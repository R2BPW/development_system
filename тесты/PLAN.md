# План автотестов

## Уровень 1 — юниты (чистые функции, без IO)

**`тесты/unit.lisp`** — SBCL + assert, без внешних зависимостей

| Функция | Тест |
|---|---|
| `%split-words` | пустая строка → nil; "a  b" → ("a" "b") |
| `%cmd-args` | "/запустить эхо задача" → "/запустить", ("эхо" "задача") |
| `%кнопка->команда` | "Породить поток" → "/породить"; "/породить длинная строка" → nil |
| `%pkg-name` | "эхо" → "ПОТОК-ЭХО" |
| `имя-пакета` (порождатель) | "(defpackage :поток-питон ...)" → "питон" |
| `извлечь-из-блока` | текст с ` ```lisp ... ``` ` → только код; без блока → текст |

## Уровень 2 — интеграция потоков (SBCL subprocess, без Telegram)

**`тесты/streams.sh`** — запускает SBCL, проверяет stdout

| Поток | Команда | Ожидаемый результат |
|---|---|---|
| `эхо` | `(выполнить "ping")` | "ping" |
| `уточнение` | `(выполнить "сделай X")` | непустая строка |
| `выполни-код` | `(выполнить "напечатай 42")` | содержит "42" |
| `реакт` | `(выполнить "2*3")` | содержит "6" |
| `порождатель` | `(выполнить "поток возвращающий привет")` | файл *.lisp создан, компилируется |

## Уровень 3 — мастер без Telegram (mock api-post)

**`тесты/master.lisp`** — загружаем :мастер, подменяем `api-post` заглушкой

| Сценарий | Вызов | Ожидание |
|---|---|---|
| Маршрутизация команд | `(обработать-команду 123 "/потоки")` | строка со списком потоков |
| Свободный текст → диалог | `(обработать-команду 123 "привет")` | непустая строка (не "Неизвестная команда") |
| `/запустить эхо тест` | `(обработать-команду 123 "/запустить эхо тест")` | "тест" |
| Несуществующий поток | `(обработать-команду 123 "/запустить нет задача")` | содержит "не найден" |
| Идемпотентность загрузки | `запустить-поток` дважды → `find-package` → одна копия |
| Admin-only фильтр | чужой chat_id → никакого ответа |

## Уровень 4 — smoke через Telegram API

**`тесты/bot.sh`** — curl + poll ответа (требует живого бота)

```bash
send_and_wait() {
  MSG_ID=$(curl -s .../sendMessage -d "chat_id=... text=$1" | jq .message_id)
  sleep 3
  curl -s .../getUpdates | jq ".result[] | select(.message.reply_to_message.message_id == $MSG_ID)"
}
```

| Команда | Ожидаемый ответ |
|---|---|
| `/старт` | непустой текст + reply keyboard |
| `/потоки` | содержит "эхо" |
| `/запустить эхо ping` | "ping" |
| `/запустить выполни-код напечатай 1+1` | "2" |
| `привет` | LLM-ответ (не "Неизвестная команда") |

## Отдельные edge cases

| Сценарий | Что проверяем |
|---|---|
| **Offset persistence** | рестарт бота → `/tmp/cl-master-offset.txt` читается → старые сообщения не дублируются |
| **Rate limit 429** | api-post получает 429 → возвращает nil → poll продолжается |
| **Broken stream** | поток с синтаксической ошибкой → лог "FAIL", остальные грузятся нормально |
| **Nil text guard** | `(send-message 123 nil)` → ничего не отправляется (нет "null" в чате) |
| **Long command** | `/породить` + 500-символьное описание → аргументы не обрезаются |

## Порядок реализации

1. `unit.lisp` — быстро, без зависимостей, запускается в CI
2. `streams.sh` — нужен OPENROUTER_API_KEY, запускать вручную
3. `master.lisp` — нужен mock api-post, средняя сложность
4. `bot.sh` — нужен живой бот + токен, только ручной прогон
