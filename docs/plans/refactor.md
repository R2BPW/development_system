## Фаза 1 — Единый контракт обработчиков

**Цель:** все `%обр-*` возвращают строку, IO только в `%dispatch`.

```lisp
;; было:
(defun %обр-старт (chat-id args)
  (send-message chat-id текст :reply-markup *главное-меню*)  ; прямой IO!
  nil)

;; стало: возвращаем (values строка &optional reply-markup)
(defun %обр-старт (chat-id args)
  (declare (ignore args))
  (values (or (душа-описание chat-id) "Мастер готов.") *главное-меню*))

;; в %dispatch:
(multiple-value-bind (текст markup) (funcall fn chat-id args)
  (when (stringp текст) (send-message chat-id текст :reply-markup markup)))
```

## Фаза 2 — Инъекция зависимостей

**Цель:** `api-post` и `llm-complete` — defvar с заглушками в тестах.

```lisp
;; telegram.lisp
(defvar *api-post-fn* #'%api-post-real)
(defun api-post (method payload) (funcall *api-post-fn* method payload))

;; llm.lisp  
(defvar *llm-fn* #'%llm-real)
(defun llm-complete (prompt &rest args) (apply *llm-fn* prompt args))
```

```lisp
;; тест-хелпер
(defmacro с-моками ((&key (api (constantly nil)) (llm (constantly "[мок]"))) &body body)
  `(let ((*api-post-fn* ,api) (*llm-fn* ,llm)) ,@body))
```

## Фаза 3 — Исправить `*llm-key*` и `*активные-потоки*`

```lisp
;; llm.lisp — ленивое чтение:
(defun %llm-key () (or *llm-key* (setf *llm-key* (uiop:getenv "OPENROUTER_API_KEY"))))

;; потоки.lisp — загрузить-все-потоки регистрирует в хеше:
(defun загрузить-все-потоки ()
  (dolist (путь (directory (merge-pathnames "*.lisp" *каталог-потоков*)))
    (when (загрузить-поток путь)
      (setf (gethash (pathname-name путь) *активные-потоки*) t))))
```

```lisp
;; история.lisp — ограничение глубины:
(defparameter *макс-история* 20)
(defun добавить-сообщение (chat-id role content)
  (let* ((история (or (загрузить-историю chat-id) '()))
         (новая (last (append история (list `((:role . ,role) (:content . ,content))))
                      *макс-история*)))
    (сохранить-историю chat-id новая) новая))
```

## Фаза 4 — Юнит-тесты (тесты/unit.lisp)

Без IO, без токенов. Запускаются в CI.

| Тест | Что проверяет |
|---|---|
| `%split-words` | пустая строка → nil; пробелы → список |
| `%cmd-args` | "/запустить эхо задача" → cmd + args |
| `%кнопка->команда` | маппинг кнопок точный |
| `%pkg-name` | "эхо" → "ПОТОК-ЭХО" |
| `душа->системный-промпт` | fixture plist → содержит имя и стиль |
| `%обр-старт` (с моком) | возвращает строку, не nil |
| `обработать-команду /потоки` (с моком) | возвращает строку |
| `обработать-команду /запустить` без аргументов | usage string |
| `добавить-сообщение` × 25 | длина ≤ `*макс-история*` |

## Фаза 6 — Тесты потоков (тесты/streams.sh)

Требует `OPENROUTER_API_KEY`. Ручной прогон.

```bash
sbcl_stream() { sbcl --load ~/quicklisp/setup.lisp \
  --load мастер/потоки/$1.lisp \
  --eval "(format t \"~A\" (поток-$1:выполнить \"$2\"))" \
  --eval "(sb-ext:exit)" 2>/dev/null; }

assert_contains "эхо"       "$(sbcl_stream эхо ping)"           "ping"
assert_contains "уточнение" "$(sbcl_stream уточнение 'сделай X')" ""   # непустой
assert_contains "выполни-код" "$(sbcl_stream выполни-код 'напечатай 42')" "42"
```

## Порядок выполнения

```
Ф1 контракт → Ф2 инъекция → Ф3 фиксы → Ф4 юниты → Ф5 интеграция → Ф6 стримы
```

Каждая фаза коммитируется отдельно. После Ф4 CI-прогон без токенов должен быть зелёным.
